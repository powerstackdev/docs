### Getting started

Here are the important steps to help you get started with contributing to Power Stack:

#### Forking the main repository

1. To contribute without being a public member you can fork the [project repository](https://github.com/powerstackdev/power-stack), see the [Forking a repository](https://docs.github.com/en/get-started/quickstart/fork-a-repo#forking-a-repository) reference
2. Clone your [forked repository](https://docs.github.com/en/get-started/quickstart/fork-a-repo#cloning-your-forked-repository) into your projects folder

#### Create a new environment based on your forked repository

3. Run the **Install Power Stack** command from the [Getting started documentation](https://powerstack.dev/docs/intro/getting-started/#install-power-stack) and replace **/powerstackdev** with your GitHub username

4. Once it has installed correctly, you will see a dashboard load in the browser (or a message in the Terminal)

5. Login by clicking **Click here to login** and use the username and password found in the message in the Terminal

#### Solve an issue

6. Go ahead and make your contributions to the project

#### Commit your update

7. When you're ready to submit your changes, stage and commit your changes. See the [Making and pushing changes](https://docs.github.com/en/get-started/quickstart/contributing-to-projects#making-and-pushing-changes) reference

#### Create a pull request

8. Head over to your forked GitGub repository and locate **Contribute** dropdown

9. Click **Open pull request**

10. GitHub will bring you to a page where you can enter a title and a description. See the [Make a pull request](https://docs.github.com/en/get-started/quickstart/contributing-to-projects#making-a-pull-request) reference

11. Click **Create pull request**

#### Managing feedback

12. Pull requests will be reviewed before merged by a verified owner

## Public members contributor guide

This section assumes you are already part of the Power Stack organisation on [GitHub](https://github.com/powerstackdev/power-stack)

#### Create a new environment

1. Run the **Install Power Stack** command from the [Getting started documentation](https://powerstack.dev/docs/intro/getting-started/#install-power-stack)

2. Once it has installed correctly, you will see a dashboard load in the browser (or a message in the Terminal)

3. Login by clicking **Click here to login** and use the username and password found in the message in the Terminal

#### Create a new issue

4. Navigate to [Issues](https://github.com/powerstackdev/power-stack/issues) and click **New issue**

5. Provide information and assign labels if required

6. Under **Development** create a new branch for this issue

7. The branch information will automatically be added by GitHub

#### Solve the issue

8. Go ahead and make your contributions to the project

#### Commit your update

9. When you're ready to submit your changes, stage and commit your changes. See the [Making and pushing changes](https://docs.github.com/en/get-started/quickstart/contributing-to-projects#making-and-pushing-changes) reference

#### Create a pull request

10. Head over to the main GitGub repository and locate **Pull requests**

11. Click on **New pull request**

11. GitHub will bring you to a page where you can enter a title and a description. See the [Make a pull request](https://docs.github.com/en/get-started/quickstart/contributing-to-projects#making-a-pull-request) reference

12. Click **Create pull request**

#### Managing feedback

13. Pull requests will be reviewed before merged by a another member
